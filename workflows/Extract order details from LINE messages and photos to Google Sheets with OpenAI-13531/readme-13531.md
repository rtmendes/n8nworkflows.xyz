# Extract order details from LINE messages and photos to Google Sheets with OpenAI

## 1. Workflow Overview

**Purpose:** Receive LINE chat messages (text or images), extract DVD order details using an OpenAI-powered agent, ask the user to confirm, and upon explicit approval append the order as a new row in Google Sheets, then reply back to the user via LINE.

**Target use cases:**
- Capturing orders submitted through LINE chat (typed messages or photos of handwritten notes)
- Reducing manual transcription into a tracking spreadsheet
- Conversational confirmation loop before committing data

### 1.1 Input Reception (LINE Webhook Trigger)
Receives inbound LINE Messaging API events (message events) and passes them into the workflow.

### 1.2 Message Routing (Text vs Image)
Routes inbound messages by `message.type`. Text proceeds directly to the AI agent; images are first downloaded from LINE and then passed to the AI agent (with binary image data).

### 1.3 AI Processing, Confirmation, and Data Entry
An AI Agent (GPT model) extracts structured fields, asks for confirmation, and only upon explicit approval calls the Google Sheets tool to append a row. The agent also produces a user-facing reply that is sent back via LINE. A short rolling memory keeps chat context per LINE user.

## 2. Block-by-Block Analysis

### Block 1 — Input Reception (LINE Webhook Trigger)

**Overview:** Accepts LINE “message” events and makes the payload available to downstream nodes (including reply token and user ID for replying and session memory).

**Nodes involved:**
- **Line Messaging Trigger1**

#### Node: Line Messaging Trigger1
- **Type / role:** `@aotoki/n8n-nodes-line-messaging.lineMessagingTrigger` — webhook trigger for LINE Messaging API events.
- **Configuration choices:**
  - Listens to events: **`message`**
  - Webhook URL is generated by n8n when the workflow is active; must be set in LINE channel settings.
- **Key fields used later:**
  - `replyToken` → used by “Reply via LINE”
  - `source.userId` → used as memory session key
  - `message.type`, `message.text`, `message.id` → used for routing and image download
- **Connections:**
  - **Output →** `Route by Message Type`
- **Edge cases / failures:**
  - Misconfigured webhook URL or LINE channel secret/access token → trigger never fires.
  - LINE retries webhooks; workflow should ideally be idempotent (this one does not include explicit deduplication).
  - Some LINE events may not include expected fields if not a standard message event (but the trigger is constrained to `message` events).

### Block 2 — Message Routing (Text vs Image)

**Overview:** Distinguishes text messages from image messages. Images are downloaded from LINE first to provide binary data to the AI agent.

**Nodes involved:**
- **Route by Message Type**
- **Download LINE Image**

#### Node: Route by Message Type
- **Type / role:** `n8n-nodes-base.switch` — branching logic based on message type.
- **Configuration choices:**
  - Two rules with renamed outputs:
    - Output key **`text`** when `{{ $json.message.type }}` equals `"text"`
    - Output key **`image`** when `{{ $json.message.type }}` equals `"image"`
- **Key expressions:**
  - `{{ $json.message.type }}`
- **Connections:**
  - **Text branch →** `AI Agent - Order Extractor1`
  - **Image branch →** `Download LINE Image`
- **Edge cases / failures:**
  - Other LINE message types (e.g., video, audio, file, sticker, location) are not handled; they will not match any rule and will effectively stop (no fallback path configured).
  - If `message.type` is missing/unexpected, no route will match.

#### Node: Download LINE Image
- **Type / role:** `@aotoki/n8n-nodes-line-messaging.lineMessagingData` — fetches message content (binary) from LINE given a message ID.
- **Configuration choices:**
  - `messageId`: `{{ $json.message.id }}`
  - Additional options: none specified
- **Inputs / outputs:**
  - **Input:** The image-message event item from the Switch node
  - **Output:** Same item plus **binary image data** (node-dependent binary field naming), which is then used by the AI agent.
- **Connections:**
  - **Output →** `AI Agent - Order Extractor1`
- **Edge cases / failures:**
  - LINE credentials missing/invalid → authentication error.
  - Message content no longer available (LINE content retention limits) → 404/failed download.
  - Large images or network timeouts.
- **Version-specific notes:**
  - This is a community node (`@aotoki/...`); its binary output field name/behavior can vary by version—verify the node’s documentation if the agent doesn’t “see” the image.

### Block 3 — AI Processing, Confirmation, and Data Entry

**Overview:** The AI agent receives either text or an image (binary), extracts order fields, asks the user for confirmation, and only on explicit approval calls the Google Sheets append operation. It then replies to the user via LINE.

**Nodes involved:**
- **AI Agent - Order Extractor1**
- **OpenAI GPT-4o1**
- **Simple Memory - Chat History1**
- **Append Row to Google Sheets**
- **Reply via LINE**

#### Node: OpenAI GPT-4o1
- **Type / role:** `@n8n/n8n-nodes-langchain.lmChatOpenAi` — chat language model provider for the agent.
- **Configuration choices:**
  - Model: **`gpt-4.1-mini`** (despite the node name mentioning GPT-4o)
  - `maxTokens`: 4096
  - `temperature`: 0.3
- **Connections:**
  - **AI language model output →** `AI Agent - Order Extractor1` (as the agent’s LLM)
- **Edge cases / failures:**
  - Missing/invalid OpenAI API key → auth error.
  - Model not available in your OpenAI account/region → model error.
  - Token limits: large images or long chat history may cause truncation or refusal.

#### Node: Simple Memory - Chat History1
- **Type / role:** `@n8n/n8n-nodes-langchain.memoryBufferWindow` — rolling window memory for conversation context.
- **Configuration choices:**
  - `sessionIdType`: custom key
  - `sessionKey`: `{{ $('Line Messaging Trigger1').item.json.source.userId }}`
  - `contextWindowLength`: 10
- **Connections:**
  - **AI memory →** `AI Agent - Order Extractor1`
- **Edge cases / failures:**
  - If `source.userId` is missing (e.g., some group/room scenarios depending on LINE settings), memory may not be stable.
  - Memory window may omit earlier corrections if conversation is long.

#### Node: Append Row to Google Sheets
- **Type / role:** `n8n-nodes-base.googleSheetsTool` — **tool node** callable by the AI Agent to append a row.
- **Configuration choices:**
  - Operation: **Append**
  - Spreadsheet (Document ID): not set in the provided JSON (must be configured)
  - Sheet name: not set (must be configured)
- **How it’s used:**
  - Connected as an **AI Tool** to the agent (not a normal “main” connection).
  - The agent is instructed to call it only after explicit user approval and to pass *only* strict JSON with these keys:
    - `game_name`, `game_date`, `customer_name`, `contact`, `delivery_address`, `dvd_quantity`
- **Connections:**
  - **AI tool connection ↔** `AI Agent - Order Extractor1`
- **Edge cases / failures:**
  - Google OAuth2 not configured or expired → auth error.
  - Spreadsheet permissions insufficient → 403.
  - Sheet name mismatch → append failure.
  - Column mapping ambiguity: if the sheet expects a different column order/headers than the tool assumes, data may be misaligned. (You should align sheet headers with the JSON fields or configure mapping if the node supports it in your n8n version.)

#### Node: AI Agent - Order Extractor1
- **Type / role:** `@n8n/n8n-nodes-langchain.agent` — orchestrates extraction, conversation, and tool calling.
- **Configuration choices:**
  - **User text input (`text` field):**
    - `{{ $json.message?.text ?? 'Please read the order information from the attached image' }}`
    - Meaning: for image messages (no `message.text`), it supplies a default instruction to read the attached image.
  - **System message (core logic):**
    - Defines “DVD order assistant”
    - Extract required fields
    - Formats a confirmation message
    - Enforces **explicit approval** before calling Google Sheets tool
    - Handles missing fields, unclear images, and cancel/stop
    - Responds in the user’s language
  - `passthroughBinaryImages`: **true** (critical for image understanding; passes binary to the model/agent pipeline)
  - `returnIntermediateSteps`: false
- **Inputs:**
  - From Switch (text) or from Download node (image with binary)
  - LLM from OpenAI node
  - Memory from Memory node
  - Tool: Google Sheets append tool
- **Outputs:**
  - Produces an `output` field (used by “Reply via LINE”)
- **Connections:**
  - **Main output →** `Reply via LINE`
  - **AI language model input ←** `OpenAI GPT-4o1`
  - **AI memory input ←** `Simple Memory - Chat History1`
  - **AI tool ←→** `Append Row to Google Sheets`
- **Edge cases / failures:**
  - If the model ignores instructions, it might attempt tool calls prematurely; tool connection still exists, so rely on prompt + (ideally) additional guardrails (not present here).
  - Date formatting: user may provide ambiguous dates; agent is expected to convert to `YYYY-MM-DD`, but could be wrong without locale assumptions.
  - If LINE image binary isn’t correctly passed/mapped, the model may respond “unclear image” or hallucinate details.

#### Node: Reply via LINE
- **Type / role:** `@aotoki/n8n-nodes-line-messaging.lineMessaging` — sends a reply message back to the user using the reply token.
- **Configuration choices:**
  - Message text: `{{ $json.output }}`
  - Reply token: `{{ $('Line Messaging Trigger1').item.json.replyToken }}`
- **Connections:**
  - **Input ←** `AI Agent - Order Extractor1`
  - No downstream nodes
- **Edge cases / failures:**
  - Reply token expires quickly; if workflow execution is slow (e.g., OpenAI latency), LINE may reject the reply.
  - If `$json.output` is empty/non-string, LINE API may error.
  - LINE credentials missing/invalid → auth error.

## 4. Reproducing the Workflow from Scratch

1. **Create a new workflow** in n8n.

2. **Add LINE Trigger**
   - Add node: **LINE Messaging Trigger** (`@aotoki/n8n-nodes-line-messaging.lineMessagingTrigger`)
   - Events: select **message**
   - Configure LINE credentials (channel access token/secret as required by the node).
   - Copy the produced **webhook URL** (Test/Production depending on your setup).

3. **Configure LINE Messaging API webhook**
   - In LINE Developers Console:
     - Set the webhook URL to the n8n trigger URL
     - Enable webhook
     - Ensure your bot has permission to receive message content

4. **Add Switch for routing**
   - Add node: **Switch** (`n8n-nodes-base.switch`)
   - Create rule #1:
     - Condition: `{{ $json.message.type }}` equals `"text"`
     - Rename output to **text**
   - Create rule #2:
     - Condition: `{{ $json.message.type }}` equals `"image"`
     - Rename output to **image**
   - Connect: **LINE Trigger → Switch**

5. **Add “Download LINE Image” node**
   - Add node: **LINE Messaging Data** (`@aotoki/n8n-nodes-line-messaging.lineMessagingData`)
   - Set **messageId** to: `{{ $json.message.id }}`
   - Connect: **Switch (image output) → Download LINE Image**

6. **Add OpenAI Chat Model node**
   - Add node: **OpenAI Chat Model** (`@n8n/n8n-nodes-langchain.lmChatOpenAi`)
   - Credentials: set your **OpenAI API key** in n8n Credentials and select it here
   - Model: choose **gpt-4.1-mini** (or your preferred compatible model)
   - Options:
     - Max tokens: **4096**
     - Temperature: **0.3**

7. **Add Memory node**
   - Add node: **Simple Memory / Buffer Window** (`@n8n/n8n-nodes-langchain.memoryBufferWindow`)
   - Session ID type: **Custom Key**
   - Session key: `{{ $('Line Messaging Trigger1').item.json.source.userId }}`
     - If your trigger node name differs, reference your trigger node by its name.
   - Context window length: **10**

8. **Add Google Sheets Tool node (append row)**
   - Add node: **Google Sheets Tool** (`n8n-nodes-base.googleSheetsTool`)
   - Credentials: configure **Google Sheets OAuth2** in n8n Credentials and select it
   - Operation: **Append**
   - Set:
     - **Spreadsheet / Document ID** (select or paste URL)
     - **Sheet name** (tab name)
   - Ensure your sheet has columns compatible with:
     - `game_name`, `game_date`, `customer_name`, `contact`, `delivery_address`, `dvd_quantity`
     - (Best practice: use these exact header names in the first row.)

9. **Add AI Agent node**
   - Add node: **AI Agent** (`@n8n/n8n-nodes-langchain.agent`)
   - Prompt type: **Define**
   - Text input:
     - `{{ $json.message?.text ?? 'Please read the order information from the attached image' }}`
   - Enable **Passthrough Binary Images** = **true**
   - System message: paste the provided “DVD order assistant” instructions (as in the workflow), ensuring it:
     - extracts the six fields
     - asks “Shall I register this order?”
     - calls Google Sheets tool only on explicit approval
     - outputs strict JSON to the tool
   - Connect resources:
     - Connect **OpenAI Chat Model → Agent** as **AI Language Model**
     - Connect **Memory → Agent** as **AI Memory**
     - Connect **Google Sheets Tool → Agent** as **AI Tool**
   - Connect message inputs:
     - **Switch (text output) → Agent**
     - **Download LINE Image → Agent**

10. **Add Reply node**
    - Add node: **LINE Messaging** (`@aotoki/n8n-nodes-line-messaging.lineMessaging`)
    - Messages: set a text message:
      - Text: `{{ $json.output }}`
    - Reply token:
      - `{{ $('Line Messaging Trigger1').item.json.replyToken }}`
    - Connect: **AI Agent → Reply via LINE**

11. **Activate the workflow**
    - Test with LINE:
      - Send a text order message
      - Send an image of a handwritten memo
    - Confirm that:
      - The bot asks for confirmation
      - Only after “OK/Yes/Register” it appends to Sheets
